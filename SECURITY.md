# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅         |
| < 1.0   | ❌         |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security vulnerability in LiqPass, please follow these steps to report it:

1. **Do not** disclose the vulnerability publicly until it has been addressed by our team.
2. Send an email to [security@liqpass.io](mailto:security@liqpass.io) with the following information:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact of the vulnerability
   - Any possible mitigations you have identified
3. We will acknowledge your report within 24 hours and provide an estimated timeline for resolution.
4. We will work with you to understand and fix the vulnerability.
5. Once the vulnerability is fixed, we will publicly disclose it in the release notes.

## Security Best Practices

### For Developers

- Always use the latest version of dependencies
- Follow secure coding practices
- Implement proper input validation
- Use secure authentication and authorization mechanisms
- Encrypt sensitive data at rest and in transit
- Regularly review code for security vulnerabilities

### For Deployments

- Keep all software up to date
- Implement proper network security controls
- Use secure configuration settings
- Monitor for suspicious activity
- Implement a robust backup strategy

## Security Audits

LiqPass undergoes regular security audits by third-party security firms. The results of these audits are available upon request for enterprise customers.

## Bug Bounty Program

We offer a bug bounty program for security researchers who discover and responsibly disclose vulnerabilities in our system. The reward amount depends on the severity of the vulnerability, as determined by the CVSS scoring system.

For more information about our bug bounty program, please contact us at [security@liqpass.io](mailto:security@liqpass.io).
